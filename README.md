# ☕ ShiftBrew — Сервис управления сменами для сети кофеен

## 📋 Описание проекта

ShiftBrew — это веб-приложение для составления расписания сотрудников сети кофеен. Система позволяет управляющему создавать смены, учитывать пожелания бариста, вести учёт рабочего времени и собирать статистику для HR и бухгалтерии.

**Цель проекта:** Создать цифровой инструмент для планирования и учёта рабочего времени, который уменьшает трудозатраты управляющих, повышает прозрачность для сотрудников и автоматически собирает статистику для HR и бухгалтерии.

## 🚀 Быстрый старт

### Вариант 1: Docker (рекомендуется)

Клонируйте репозиторий:
git clone https://github.com/kristz11/shiftbrew.git
cd shiftbrew

Запустите приложение:
docker-compose up -d

Приложение доступно:
- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs

### Вариант 2: Локальная установка

Backend:
cd backend
python -m venv venv
source venv/bin/activate  (Windows: venv\Scripts\activate)
pip install -r requirements.txt
uvicorn main:app --reload

Frontend:
cd frontend
npm install
npm start

## 👥 Роли пользователей

### Управляющий (Manager)
- Создание и редактирование расписания
- Просмотр расписания всех сотрудников
- Учёт пожеланий сотрудников при составлении графика
- Статистика по отработанным часам
- Экспорт данных для бухгалтерии
- Одобрение/отклонение запросов на замену смен

### Администратор (Admin)
- Полный доступ ко всем функциям системы
- Управление пользователями и кофейнями

### Бариста (Barista)
- Просмотр личного расписания
- Подача пожеланий по сменам (доступные/недоступные дни)
- Запрос на замену смены
- Отслеживание медосмотров и аттестаций
- Статистика по отработанному времени

### Стажёр (Trainee)
- Ограниченный доступ к расписанию
- Отслеживание прогресса аттестаций
- Информация о предстоящих медосмотрах

## 📦 Функциональность

### Модуль 1: Составление и редактирование расписания
- Создание смен с указанием даты, времени, роли и кофейни
- Редактирование существующих смен
- Удаление смен
- Автоматическая проверка конфликтов

### Модуль 2: Просмотр расписания
- Управляющий видит расписание всех сотрудников
- Бариста видит своё расписание и коллег по смене
- Фильтрация по датам и кофейням

### Модуль 3: Учёт часов и отработанного времени
- Автоматический подсчёт отработанных часов
- Учёт сверхурочных часов
- Статистика за период (день/неделя/месяц)
- Экспорт данных для HR и бухгалтерии

### Модуль 4: Уведомления и замены смен
- Подача пожеланий по сменам
- Запрос на замену смены
- Одобрение/отклонение запросов
- История изменений расписания

### Модуль 5: Личный кабинет сотрудника
- Информация о медосмотрах (дата последнего, следующего)
- Уровень аттестации
- Статус сотрудника (активен/в отпуске/на больничном)
- Дата приёма на работу

## 🛠 Технологии

### Backend
- Python 3.11+ — язык программирования
- FastAPI — веб-фреймворк

- JWT (python-jose) — аутентификация и авторизация

### Frontend
- React 18 — библиотека для создания пользовательских интерфейсов
- React Router — маршрутизация
- Axios — HTTP-клиент для API запросов
- Date-fns — работа с датами

### DevOps
- Docker — контейнеризация
- Docker Compose — оркестрация контейнеров
- Nginx — веб-сервер для frontend

## 📚 API Документация

После запуска приложения документация API доступна по адресам:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

Основные эндпоинты:

Аутентификация:
- POST /token — получение JWT токена
- POST /register — регистрация нового пользователя
- GET /users/me — получение информации о текущем пользователе

Смены:
- POST /shifts — создание смены (manager/admin)
- GET /shifts/my — получение моих смен
- GET /shifts/coffee-shop/{shop_id} — получение смен кофейни
- DELETE /shifts/{shift_id} — удаление смены (manager/admin)

Пожелания:
- POST /shift-wishes — создание пожелания
- GET /shift-wishes/my — получение моих пожеланий

Запросы на замену:
- POST /shift-requests — создание запроса на замену
- GET /shift-requests/my — получение моих запросов
- PUT /shift-requests/{request_id}/{status} — обновление статуса запроса

Статистика:
- GET /work-time/my-stats — статистика по моему времени
- GET /work-time/stats — общая статистика (manager/admin)

##  Сценарии использования

### Сценарий 1: Создание расписания
1. Управляющий входит в систему
2. Переходит в раздел "Расписание"
3. Нажимает "Создать смену"
4. Заполняет форму (дата, время, сотрудник, кофейня)
5. Система сохраняет смену и уведомляет сотрудника

### Сценарий 2: Подача пожелания
1. Бариста входит в систему
2. Переходит в раздел "Мои пожелания"
3. Выбирает дату и тип пожелания (не могу работать/предпочитаю утро)
4. Добавляет комментарий (опционально)
5. Отправляет пожелание
6. Управляющий видит пожелание при составлении расписания

### Сценарий 3: Замена смены
1. Бариста не может выйти на смену
2. Создаёт запрос на замену, указывая коллегу
3. Указывает причину
4. Управляющий получает уведомление
5. Управляющий одобряет запрос
6. Оба сотрудника получают уведомление об изменении
